



### General Concepts
**1. What are the similarities and differences between (1) containers, (2) sequences, and (3) mappings?** 

Containers are any objects that hold other objects. Sequences are a specific type of container that stores items in a fixed, ordered, linear arrangement accessed by index (0, 1, 2...). Mappings store items as key-value pairs and are accessed by a unique key rather than an index. 

**2. All sequences are containers, but not all containers are sequences. Explain, with an example of a container that is not a sequence.** 

A container simply holds objects, while a sequence implies a specific ordered structure. A dictionary (dict) is a container because it holds items, but it is not a sequence because it uses keys (not index positions) to access data. 

**3. Strings are "immutable" while lists are "mutable." Explain what this means and why it matters when writing code.** 

"Immutable" means you cannot change an object's contents after it is created; "mutable" means you can change it in place. This matters because mutable objects (like lists) can be modified directly, while immutable objects (like strings) require you to create a new object if you want to "change" them. 

**4. What is a negative index in Python? Explain using the rule: `seq[-n]` is the same as `seq[len(seq) - n]`.** 

A negative index allows you to access elements starting from the end of a sequence. An index of `-1` refers to the last item, `-2` the second-to-last, and so on. 

**5. All sequence types are iterable — explain what this means and how the container/sequence/mapping distinction relates to it**. 

"Iterable" means you can loop through the object’s elements one by one (e.g., using for item in container). All sequences and mappings are iterables, meaning you can traverse their contents, even though they use different methods for access (index vs. key). 

**6. The '+' operator can be used to concatenate two sequences of the same type. Explain, and describe how this is an example of operator overloading.** 

The `+` operator joins two sequences together into a new, combined sequence. This is "operator overloading" because the + symbol behaves differently depending on the data type—it adds numbers, but it joins sequences. 

**7. An empty list evaluates to False in a Boolean context. Explain this rule, and state whether the contents of a non-empty list ever affect its truth value.** 

Python considers an empty list to be `False` and any list with at least one item to be `True`. The contents do not matter; even a list containing `[0]` or `[None]` is considered True because the list is not empty. 

### Slicing

**8. A slice of seq from i to j with step k is defined as the sequence of items at index `x = i + n*k`, `for 0 <= n < (j-i)/k`. Explain this in plain language with a worked example.** 

This formula defines which items to pick when slicing with a step. If you have `my_list[0:6:2]`, it starts at index `0`, and then adds `2` repeatedly `(0, 2, 4)` until it reaches the limit 6.

### Assignment, Copying, and Aliasing

**11. What is the difference between assigning a list to another variable and copying a list?** 

Assigning (`a = b`) creates an alias, where both variables point to the same object in memory. Copying (using `[:]` or `.copy()`) creates a brand-new object that is independent of the original. 

**12. What is "aliasing"? Why can aliasing a mutable object like a list cause unexpected bugs? Give an example.** 

Aliasing is when two variables point to the same mutable object. If you modify one variable, the other changes too, which often causes hard-to-find bugs. 
```python
list1 = [1, 2]
list2 = list1  # Aliasing
list2.append(3)
print(list1)   # Output: [1, 2, 3] - unexpected!
```


**13. Both `copy_list = original_list[:]` and `copy_list = original_list.copy()` produce a copy. Are they equivalent? Explain.** 

Yes, they are both standard ways to perform a "shallow copy," creating a new list object with the same items. 

**14. Explain why `my_matrix = [[0] * 3] * 4` does NOT create four independent rows. Write a script that proves this by modifying one row, then fix it using list comprehension so the rows are independent.** 

The `* 4` operator repeats the reference to the same row object four times. 

```python
# Bad way
matrix = [[0] * 3] * 4
matrix[0][0] = 9
print(matrix) # [[9, 0, 0], [9, 0, 0], ...] (All rows changed)

# Fixed way
matrix = [[0] * 3 for _ in range(4)]
matrix[0][0] = 9
print(matrix) # [[9, 0, 0], [0, 0, 0], ...] (Only one row changed)
```

**15. What is a "deep copy"? Using the copy module, write a script showing that `copy.deepcopy()` solves the problem in Q14, while a plain `.copy()` does not.** 

A deep copy recursively copies nested objects, ensuring they are not shared. 
```python
import copy
nested = [[1, 2], [3, 4]]
shallow = nested.copy()
deep = copy.deepcopy(nested)

shallow[0][0] = 9
print(nested) # [[9, 2], [3, 4]] (nested modified)
deep[0][0] = 9
print(nested) # [[9, 2], [3, 4]] (nested unchanged)
```

**29. Remove duplicates while preserving order.**

def remove_dupes(lst):
    seen = set()
    result = []
    for x in lst:
        if x not in seen:
            result.append(x)
            seen.add(x)
    return result

**30. Write a function that removes duplicates from a list and returns two lists: one with each item appearing once, and one containing only the items that were duplicated.**

```python
def split_dupes(lst):
    uniques, seen, duplicates = [], set(), set()
    for x in lst:
        if x not in seen:
            uniques.append(x)
            seen.add(x)
        else:
            duplicates.add(x)
    return uniques, list(duplicates)
```

**34. Write a function that merges two sorted lists (which may be of different lengths, and either may be empty) into one sorted list. Do not use any built-in sort/merge functions.**

```python
def merge(l1, l2):
    res = []
    i = j = 0
    while i < len(l1) and j < len(l2):
        if l1[i] < l2[j]: res.append(l1[i]); i += 1
        else: res.append(l2[j]); j += 1
    res.extend(l1[i:])
    res.extend(l2[j:])
    return res
```

**36. A polynomial `p(x) = -13.39 + 17.5x + 3x² + x⁴` can be represented as `[-13.39, 17.5, 3, 0, 1.0]`, where the index represents the power of x and the value at that index is the coefficient. Write a function to evaluate the polynomial for a given x.**
