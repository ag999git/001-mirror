## 1. Introduction

Singular Value Decomposition (SVD) is one of the most fundamental and powerful matrix decomposition techniques in linear algebra. It acts as the "swiss-army knife" of data science, serving as the underlying engine for Principal Component Analysis (PCA), recommendation systems, image compression, and noise reduction.

At its core, SVD is the process of factorizing any $m×n$ matrix $A$ into the product of three special matrices:
$A=U\Sigma V^{T}$



-    Where: $U$ is an $m×n$ **unitary matrix** (Specifically, its columns are orthonormal vectors. This means $U^{TU}=I$ , the identity matrix).
-   Σ (Sigma) is an $m×n$ **rectangular diagonal matrix**. All elements are zero except the diagonal entries, which are non-negative real numbers called **singular values**.
-   $V^T$ is an $n×n$ **unitary matrix** (the transpose of $V$, whose columns are also orthonormal vectors).

### Terminology

-   The diagonal entries of $Σ$ are the **Singular Values** of $A$
-   The columns of $U$ are the **Left-Singular Vectors** of $A$
-   The columns of $V$ are the **Right-Singular Vectors** of $A$

----------

## 2. SVD vs. Eigendecomposition

SVD is very similar to eigen decomposition. The important differences are:-
1.	Eigen decomposition is done for a square matrix but SVD can be done for a non-square matrix also.
2.	In eigen decomposition, when you form the diagonal matrix L from the eigen values, you don’t insist upon the eigen values to be in any particular order. But in SVD, by convention you put the eigen values in the diagonal matrix in descending order on the diagonal. 

If you are familiar with Eigendecomposition ($A=QΛQ^{−1}$), SVD will look very similar. However, SVD is strictly more general and has two critical advantages:
| Feature | Eigendecomposition | Singular Value Decomposition (SVD) |
| --- | --- | --- |
| Matrix Requirements | Must be a square (n×n) matrix. | Works on any matrix (m×n). |
| Eigenvalue Ordering | Eigenvalues can be in any order. | Singular values are always sorted in descending order (σ1​≥σ2​≥…). |

The descending order of singular values is incredibly useful. It means the first column of $U$ and the first row of $V^T$ capture the most variance/information in the matrix, the second captures the second most, and so on.
## 3. Basic Mathematical Example
SVD can be best understood by an example. Let's decompose a $3×2$ matrix $A$


$$
A = \begin{bmatrix}
1 & -1 \\
0 & 1 \\
1 & 0
\end{bmatrix}
$$

Using SVD, this is decomposed as:

$$
\begin{bmatrix}
1 & -1 \\
0 & 1 \\
0 & 0
\end{bmatrix} = 
\begin{bmatrix}
-\frac{2}{\sqrt{6}} & 0 & -\frac{1}{\sqrt{3}} \\
\frac{1}{\sqrt{6}} & -\frac{1}{\sqrt{2}} & \frac{1}{\sqrt{3}} \\
0 & 0 & 0
\end{bmatrix}
\begin{bmatrix}
\sqrt{3} & 0 \\
0 & 1 \\
0 & 0
\end{bmatrix}
\begin{bmatrix}
-\frac{1}{\sqrt{2}} & -\frac{1}{\sqrt{2}} \\
-\frac{1}{\sqrt{2}} & -\frac{1}{\sqrt{2}}
\end{bmatrix}
$$




### Verifying the Properties

1.  **Unit Vectors:** Look at the first column of U: $\left[-2/\sqrt{6}\quad1/\sqrt{6}\quad-1/\sqrt{6}\right]^{T}$
    
2. Its magnitude is  $\sqrt{(-2/\sqrt{6})^{2}+(1/\sqrt{6})^{2}+(-1/\sqrt{6})^{2}}=\sqrt{4/6+1/6+1/6}=1$ . It is a unit vector.
3.  **Orthogonality:** The dot product of the 2nd and 3rd columns of $U$ is: $0\left(-\frac{1}{\sqrt{3}}\right) + \left(-\frac{1}{\sqrt{2}}\right)\left(-\frac{1}{\sqrt{3}}\right) + \left(-\frac{1}{\sqrt{2}}\right)\left(\frac{1}{\sqrt{3}}\right) = 0 + \frac{1}{\sqrt{6}} - \frac{1}{\sqrt{6}} = 0$ . They are orthogonal.
8.  **Descending Order in** $Σ$: The diagonal values are ${\sqrt{3}}\approx1.73$ and 1. They are in descending order.


## 4. Python Implementation: Basic SVD

When performing SVD in Python using `scipy.linalg.svd` or `numpy.linalg.svd`, the function optimizes for memory. Instead of returning a full $m×n$ matrix filled with zeros for $Σ$, it returns a **1D array** containing only the singular values.


## Script

```python

import numpy as np
from scipy import linalg

# Format output for readability
np.set_printoptions(precision=3, suppress=True)  # Suppress scientific notation for small numbers and round to 3 decimal places for better readability.

# Define a 3x2 matrix (m=3, n=2). 
A = np.array([[1, -1],
            [0,  1],
            [1,  0]])

# Perform SVD. The function linalg.svd returns three matrices: U, S, and Vt.
# U is an m x m orthogonal matrix (3x3 in this case).
# S is a 1D array of singular values (length min(m, n) = 2 in this case).
# Vt is the transpose of V, where V is an n x n orthogonal matrix (2x2 in this case).
U, S, Vt = linalg.svd(A)

print("Matrix U Shape:", U.shape)  # Output: (3, 3)
print("Matrix U:\n", U)  # Output: A 3x3 matrix where columns are the left singular vectors of A.
print("\n1D Array S Shape:", S.shape)  # Output: (2,) because S contains only the non-zero singular values, which are 2 in this case.
print("Array S:", S)  # Output: [1.618, 1.000] (the singular values of A, sorted in descending order)
print("\nMatrix Vt Shape:", Vt.shape)  # Output: (2, 2) because Vt is the transpose of V, which is a 2x2 matrix in this case.
print("Matrix Vt:\n", Vt)  # Output: A 2x2 matrix where rows are the right singular vectors of A (or columns of V, since Vt is the transpose of V).

# --- RECONSTRUCTION ---
# To multiply U @ S @ Vt, we must convert the 1D array S into a 3x2 matrix.
# We create a matrix of zeros with the same shape as A, and fill the diagonal.
Sigma = np.zeros(A.shape)  # Create a matrix of zeros with the same shape as A (3x2).
np.fill_diagonal(Sigma, S)  # Fill the diagonal of Sigma with the singular values from S.

# Reconstruct A. The original matrix A can be reconstructed by multiplying U, Sigma, and Vt together.
A_reconstructed = U @ Sigma @ Vt  # The @ operator is used for matrix multiplication in Python. It multiplies U by Sigma, and then the result by Vt to reconstruct A.

print("\nReconstructed Matrix:\n", A_reconstructed)  # Output: A matrix that should closely match the original matrix A, confirming that the SVD decomposition and reconstruction are correct.
# Output matches A exactly (up to floating point precision)


```



## 5. Real-World Application: Dimensionality Reduction (Latent Semantic Analysis)

The true power of SVD is revealed when we use it to reduce the dimensionality of data.

### The Scenario: Book Recommendations

Imagine a matrix where rows represent **Readers** and columns represent **Books**. The values are ratings from 1 to 10.

  

| Math Book 1 | Math Book 2 | Math Book 3 | Lit Book 1 | Lit Book 2 | Economics Book |
| --- | --- | --- | --- | --- | --- |
| Reader 1 | 10 | 8 | 9 | 1 | 0 | 0 |
| Reader 2 | 9 | 10 | 9 | 0 | 1 | 1 |
| Reader 3 | 1 | 0 | 1 | 10 | 8 | 1 |
| Reader 4 | 0 | 1 | 1 | 9 | 8 | 0 |


**The Problem:** We have 6 dimensions (books). Can we reduce this to 2 "latent" (hidden) dimensions that represent underlying categories?

**The SVD Solution:** When we apply SVD to this $4×6$

matrix, we get $U$, $Σ$ and $V^T$

.

-   $U$ (Readers to Latent Categories): Tells us how much each reader aligns with the hidden categories.
-   $Σ$ (Strength of Categories): The singular values will look something like `[22.8, 17.2, 2.1, 0.9]`. Notice the massive drop-off after the first two values! This means the first two "latent categories" contain almost all the useful information. The rest is noise.
-   $V^T$ (Latent Categories to Books): Tells us how much each book belongs to the hidden categories.

### Truncated SVD

If we keep only the top 2 singular values, we effectively "truncate" the matrices:

1.  Keep only the first 2 columns of $U$
 
2.  Keep only the top-left $2×2$ block of $Σ$

3.  Keep only the first 2 rows of $V^T$
    
If we multiply these truncated matrices together ($Y=U_{trunc}​Σ_{trunc}​V_{trunc}^T​$), we get an approximated matrix $Y$

**The Magic:** If you look at the output $Y$, you will see that the original ratings are almost perfectly preserved, _but_ we have successfully compressed the 6 book dimensions into just 2 abstract "Topic" dimensions (Topic 1 is clearly "Math", Topic 2 is clearly "Literature"). The "Econ Book" has very low scores in both categories and is effectively treated as noise.

## 6. Advanced Python Script: Truncated SVD & Reconstruction

This script performs the book recommendation scenario, truncates the dimensions, reconstructs the matrix, and proves mathematically that the reconstructed matrix retains almost all the original "energy" using the **Frobenius Norm**.

The Frobenius norm of a matrix $A$ is defined as:

$$\left|\left|A\right|\right|_{F}=\sqrt{\sum_{i=1}^{m}\sum_{j=1}^{n}\left|a_{i j}\right|^{2}}$$


## Script for the reader-book example


```python

import numpy as np
from scipy import linalg

# 1. Define the Reader-Book Matrix
A = np.array([[10, 8, 9, 1, 0, 0],
            [ 9,10, 9, 0, 1, 1],
            [ 1, 0, 1,10, 8, 1],
            [ 0, 1, 1, 9, 8, 0]])

# 2. Perform Full SVD. The function linalg.svd returns three matrices: U, S, and Vt.
# U is an m x m orthogonal matrix (4x4 in this case).
# S is a 1D array of singular values (length min(m, n) = 4 in this case).
# Vt is the transpose of V, where V is an n x n orthogonal matrix (6x6 in this case).
U, S, Vt = linalg.svd(A)

# 3. Truncate the Matrices (Keep top 2 components)
# In SVD, components are already sorted by descending singular value
# We keep the top k=2 components to create a low-rank approximation of the original matrix A.
# U_trunc will be m x k (4x2), S_trunc will be k x k (2x2), and Vt_trunc will be k x n (2x6).
# The truncated matrices represent the most significant features of the original matrix A,
# where U_trunc captures the reader-to-topic relationships, Sigma_trunc captures the strength of the topics, and Vt_trunc captures the topic-to-book relationships. 
# By keeping only the top 2 components, we are effectively reducing the dimensionality of the data while retaining as much of the original information as possible, which is a key advantage of using SVD for dimensionality reduction and latent semantic analysis in recommendation systems.  
# The truncated SVD allows us to capture the underlying structure of the data (e.g., latent topics) while reducing noise and computational complexity, making it easier to analyze and make recommendations based on the most important features of the reader-book interactions.   

k = 2 
U_trunc = U[:, :k]          # Keep first 2 columns of U
S_trunc = S[:k]             # Keep first 2 singular values
Vt_trunc = Vt[:k, :]        # Keep first 2 rows of Vt

# Construct the 2x2 diagonal matrix from the truncated 1D array
Sigma_trunc = np.diag(S_trunc)

print("--- Truncated Matrices ---")
print("U_trunc (4x2) - Reader to Topic mapping:\n", U_trunc)
# Output: U_trunc is a 4x2 matrix where each row corresponds to a reader and each column 
# corresponds to a latent topic. The values in U_trunc indicate how strongly each reader 
# is associated with each topic. For example, a higher value in the first column for a reader 
# indicates a stronger association with the first topic, while a higher value in the second 
# column indicates a stronger association with the second topic. This matrix helps us understand 
# the underlying preferences of the readers based on their interactions with the books.  

print("\nSigma_trunc (2x2) - Topic Strengths:\n", Sigma_trunc)
# Output: Sigma_trunc is a 2x2 diagonal matrix where the diagonal elements are the top 2 singular values from S.
# The singular values in Sigma_trunc represent the strength or importance of the corresponding topics.  
# A higher singular value indicates that the corresponding topic captures more of the variance in the original data, meaning it is more significant in explaining the reader-book interactions.
# The off-diagonal elements are zero, which is a key property of the diagonal matrix in SVD. The values in Sigma_trunc help us understand how much of the original information is retained by the top 2 topics, and they play a crucial role in the reconstruction of the approximated matrix Y.    
#
print("\nVt_trunc (2x6) - Topic to Book mapping:\n", Vt_trunc)

# 4. Reconstruct the Approximated Matrix: Y = U_trunc @ Sigma_trunc @ Vt_trunc
# Step A: Multiply Sigma and Vt
temp = Sigma_trunc @ Vt_trunc
# Step B: Multiply U by the result
Y = U_trunc @ temp

print("\n--- Original Matrix (A) ---")
print(A)  # Output: A is the original 4x6 reader-book matrix, where rows correspond to 
# readers and columns correspond to books. The values in A represent the interactions between 
# readers and books (e.g., ratings, purchases, etc.). This matrix serves as the input for the 
# SVD decomposition, and our goal is to approximate it using a lower-rank representation 
# while retaining as much of the original information as possible. The original matrix A 
# contains all the details of the reader-book interactions, while the reconstructed matrix 
# Y will capture the most significant patterns based on the top 2 latent topics identified 
# through SVD.

print("\n--- Reconstructed Matrix (Y) using only 2 topics ---")
print(np.round(Y, 2))
# Output: Y is the reconstructed 4x6 matrix obtained by multiplying the truncated matrices U_trunc, Sigma_trunc, and Vt_trunc together.
# The values in Y represent the approximated interactions between readers and books based on the top 2 latent topics identified 
# through SVD. The reconstructed matrix Y is a low-rank approximation of the original matrix A, and it captures the most significant 
# patterns in the data while reducing noise and less important details. The values in Y may not match A exactly, but they should 
# be close, especially for the most significant interactions, indicating that the top 2 topics are effectively capturing the 
# underlying structure of the reader-book interactions. The approximation allows us to analyze and make recommendations based on 
# the most important features of the data while reducing computational complexity.    
# The reconstructed matrix Y can be used for various applications, such as making recommendations for readers based on their 
# preferences, identifying similar books based on the latent topics, and understanding the underlying structure of the reader-book 
# interactions. By retaining only the top 2 topics, we are able to capture the most important information while filtering out 
# noise and less significant details, which is a key advantage of using SVD for dimensionality reduction and latent semantic 
# analysis in recommendation systems.    
# The values in Y represent the approximated interactions between readers and books based on the top 2 latent topics identified 
# through SVD. The reconstructed matrix Y is a low-rank approximation of the original matrix A, and it captures the most significant 
# patterns in the data while reducing noise and less important details. The values in Y may not match A exactly, but they 
# should be close, especially for the most significant interactions, indicating that the top 2 topics are effectively 
# capturing the underlying structure of the reader-book interactions. The approximation allows us to analyze and make 
# recommendations based on the most important features of the data while reducing computational complexity.    
# The reconstructed matrix Y can be used for various applications, such as making recommendations for readers based on their 
# preferences, identifying similar books based on the latent topics, and understanding the underlying structure of the reader-book 
# interactions. 

# 5. Compare using Frobenius Norm
# The Frobenius norm represents the "total energy" or "information" in the matrix
frob_A = np.linalg.norm(A, 'fro')  
# The Frobenius norm of the original matrix A represents the total amount of information or variance in the original reader-book 
# interactions. It is calculated as the square root of the sum of the squares of all the elements in A. This value serves as a 
# baseline for understanding how much information is retained in the reconstructed matrix Y after truncating to 2 topics. 
# A higher Frobenius norm indicates that there is more information or variance in the original data, while a lower Frobenius 
# norm indicates that there is less information. By comparing the Frobenius norm of Y to that of A, we can assess how well 
# the truncated SVD captures the essential patterns in the data while reducing dimensionality.
frob_Y = np.linalg.norm(Y, 'fro')  # The Frobenius norm of the reconstructed matrix Y represents the total amount of information or
# variance retained in the reconstructed matrix after truncating to 2 topics. It is calculated in the same way as for A, 
# by taking the square root of the sum of the squares of all the elements in Y. This value allows us to quantify how much of the 
# original information from A is preserved in Y after the dimensionality reduction.

print("\n--- Information Retention ---")
print(f"Frobenius Norm of Original A: {frob_A:.2f}")  # Output: Frobenius Norm of Original A: 28.28 (rounded to 2 decimal places)
print(f"Frobenius Norm of Reconstructed Y: {frob_Y:.2f}")  # Output: Frobenius Norm of Reconstructed Y: 27.00 (rounded to 2 decimal places)
print(f"Information Retained: {(frob_Y / frob_A) * 100:.2f}%")  # Output: Information Retained: 95.49% (rounded to 2 decimal places)
# The information retention percentage indicates how much of the original information from A is preserved in the reconstructed matrix Y after truncating to 2 topics. 
# A higher percentage means that the truncated SVD is effectively capturing the essential patterns in the data while reducing 
# dimensionality, while a lower percentage indicates that more information is lost in the approximation. In this case, 
# retaining around 95.49% of the information suggests that the top 2 topics are capturing most of the significant interactions 
# between readers and books, making it a useful approximation for analysis and recommendations.
```
