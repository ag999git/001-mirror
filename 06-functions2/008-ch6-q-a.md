### **Answer**
- **Call by Value (for immutable objects):**
  - Immutable objects: `int`, `float`, `str`, `tuple`
  - Modifications inside the function do **not** affect the original value

- **Call by Reference (for mutable objects):**
  - Mutable objects: `list`, `dict`, `set`
  - Changes inside the function affect the original object directly

### 20\. Trapezoidal rule numerical integration (Advance usage)
b. This exercise requires knowledge of Trapezoidal rule of numerical integration. The value of a definite integral (ie the area under the function) can be found using numerical integration. One common method for this is the Trapezoidal rule given by:-

$$
\text{Area} = \int_{a}^{b} f(x) \, dx \approx h \left( \frac{1}{2} (f(a) + f(b)) + \sum_{i=1}^{n-1} f(a + ih) \right) \quad \text{where } h = \frac{b-a}{n}
$$

Write a function which takes the following 4 parameters:-
-	function f
-	lower limit a
-	upper limit b
-	number of intervals n  
Also write a docstring for the function which says:-  '''Calculates the definite integral of a function f(x), between the boundaries a, b, by dividing the area to n equal trapezoids/ strips'''
A sample program which calculates

$$
\int_{1}^{2} \left(\frac{1}{x}\right) dx
$$

is shown below. 
We know that:-

$$
\int_{1}^{2} \left(\frac{1}{x}\right) dx = \ln(2) - \ln(1) = \ln(2)
$$



### Answer

```python
from math import log

def trapez_integrate(f, a, b, n):
    """
    Calculates the numerical value of the definite integral of
    a function f(x) using the Trapezoidal Rule.

    Args:
        f (function): The function to integrate.
        a (float): The lower limit of integration.
        b (float): The upper limit of integration.
        n (int): The number of trapezoidal strips (intervals).
    
    Returns:
        float: The approximate area under the curve.
    """
    # 1. Calculate the width of each strip (h)
    h = (b - a) / n
    
    # 2. Initialize the area sum with the average of the endpoints (0.5 * f(a) + 0.5 * f(b))
    # This is equivalent to (1/2)*(f(a) + f(b)) in the formula.
    y_sum = (f(a) + f(b)) / 2
    
    # 3. Sum the function values at the intermediate points (f(a + i*h))
    # Iterate from i=1 up to i=n-1
    for i in range(1, n):
        x_i = a + i * h
        y_sum += f(x_i)
        
    # 4. Multiply the total sum by the strip width (h) to get the final area
    area = h * y_sum
    return area

# --- Application to the problem: Integral of 1/x from 1 to 2 ---

def f_x(x):
    """The function f(x) = 1/x"""
    if x == 0:
        raise ValueError("Function is undefined at x=0")
    return 1 / x

# Define parameters for the integral
a = 1.0  # Lower limit
b = 2.0  # Upper limit
n = 10000 # Number of strips (higher 'n' yields better accuracy)

# Calculate the area using the Trapezoidal Rule
area_approx = trapez_integrate(f_x, a, b, n)

# Calculate the exact analytical value for comparison
area_exact = log(2) 

print(f"--- Trapezoidal Rule for Integral of 1/x from {a} to {b} ---")
print(f"Number of strips (n): {n}")
print(f"Approximate Area (Trapezoidal): {area_approx:.10f}")
print(f"Exact Area (ln(2)):{area_exact:.10f}")
print(f"Difference:{abs(area_exact - area_approx):.10f}")

```

### Linear Congruential Generator
Linear Congruential Generator:- Python has modules for implementing Pseudo Random Number Generator (PRNG). One of the oldest PRNG algorithms is the Linear Congruential Generator. The goal is to implement the Linear Congruential Generator (LCG) algorithm to produce a sequence of pseudorandom numbers. It is given as:- 
$X_{n+1} = (a \cdot X_n + c) \bmod m$.

### Solution
The given version addresses uses a class to store the state (new_seed). This makes the generator self-contained, thread-safe, and object-oriented (more Pythonic).

```python
import math

class LinearCongruentialGenerator:
    """
    Implements the Linear Congruential Generator (LCG) PRNG algorithm.
    Uses typical parameters: m = 2^31, a = 1103515245, c = 12345.
    """
    def __init__(self, seed=1):
        # Parameters from the problem description
        self.multiplier = 1103515245
        self.increment = 12345
        self.modulo = 2**31
        
        # Initial state (seed)
        self.seed = seed

    def get_lcg(self):
        """
        Calculates the next pseudorandom value, updates the internal state,
        and returns the raw integer result X_n+1.
        """
        # X_n+1 = (a * X_n + c) mod m
        new_value = (self.multiplier * self.seed + self.increment) % self.modulo
        self.seed = new_value  # Update state for the next call
        
        # Return the raw pseudorandom integer
        return new_value
    
    def get_random(self):
        """
        Generates and scales the result to a float between [0, 1).
        """
        raw_int = self.get_lcg()
        # Scale to [0, 1) by dividing by the modulo
        return raw_int / self.modulo

# --- Usage Example ---
# 1. Create a generator object (seeds it with 123456)
lcg_gen = LinearCongruentialGenerator(seed=123456)

print("--- LCG Output (Raw Integers) ---")
print(f"Seed: {lcg_gen.seed}")
print(f"X_1: {lcg_gen.get_lcg()}")
print(f"X_2: {lcg_gen.get_lcg()}")
print(f"X_3: {lcg_gen.get_lcg()}")

print("\n--- LCG Output (Scaled to [0, 1)) ---")
lcg_gen_2 = LinearCongruentialGenerator(seed=123)
print(f"Random 1: {lcg_gen_2.get_random()}")
print(f"Random 2: {lcg_gen_2.get_random()}")

```

### Assignment
In python we can do what may be called the “Partial unpacking of iterables”. This is shown in the code below:-

Partial unpacking of iterables in Python is a feature that allows you to assign the remaining, unassigned elements of an iterable (like a list or tuple) to a single variable using the asterisk (*) operator.

It's an extension of the standard tuple/list unpacking, allowing for variable assignment when the number of elements in the iterable is unknown or greater than the number of variables specified.

#### Explaining Partial Unpacking
#### 1. The Role of the Asterisk (*)
In this context, the asterisk (*) is often called the "catch-all" or "star expression".

Catch-All Variable:- When placed before a variable name during unpacking (e.g., *rest), that variable collects all elements in the iterable that were not assigned to the regular variables.

Result Type:- The variable prefixed with the asterisk will always be assigned a list, even if the original iterable was a tuple or if it collected zero elements.

Key Constraint:- You can only use one starred expression (*variable) in any single unpacking operation. Python would not know how to distribute the remaining items if you used two.

#### 2. Positioning the Asterisk
The asterisk can be placed at any position among the variables, not just at the end.

##### Example 1:- Catching the Tail (Most Common) The variable with the asterisk collects all elements from its position to the end.

```python
first, second, *tail = [10, 20, 30, 40, 50]
print(first)  # 10
print(second) # 20
print(tail)   # [30, 40, 50]

```
##### Example 2: Catching the Middle The variables without the asterisk consume elements from the beginning and end, and the starred variable collects everything in between.

```python
start, *middle, last = [1, 2, 3, 4, 5, 6]
print(start)  # 1
print(middle) # [2, 3, 4, 5] 
print(last)   # 6

```
### Assignment
Concept: Brute-Force String SearchingThe Brute-Force method is the simplest, most intuitive approach to finding a pattern $P$ of length $n$ within a larger text $T$ of length $m$.It works by checking every possible alignment of $P$ against $T$.

#### Analogy
- Imagine you are looking for a four-letter word ("C-A-T-S") in a long sentence (the Text).
- The Brute-Force approach is like using a four-finger stencil (the Pattern).
- You place the stencil over the first four letters of the sentence.
- You check all four letters at the same time.
- As soon as one letter doesn't match, you immediately move the stencil one position to the right and start checking the new four letters from the beginning. You don't try to reuse any information from the previous check.
- This is simple but means you sometimes re-check the same characters many times (the inefficiency).



#### The algorithm is implemented as follows

1. Alignment (Outer Loop): The algorithm tries to align the start of the pattern $P$ with every character in the text $T$, from index $i=0$ up to index $i=m-n$.
2. Comparison (Inner Loop): For each alignment $i$, it compares the characters of the pattern $P$ (from index $x=0$ to $n-1$) with the corresponding characters in the text $T$ ($T[i+x]$).
3. Shifting (Sliding): If a mismatch occurs at any point during the comparison, the pattern is immediately shifted (slid) one position to the right ($i$ is incremented), and the comparison restarts from the beginning of the pattern.
4. Success: If the comparison completes successfully (the inner loop finishes without finding a mismatch, i.e., $x$ reaches $n$), a match is found, and the starting index $i$ is returned.

```python

def brute_force_search(text, pattern):
    '''
    Implements the Brute-Force string searching algorithm.
    
    Parameters:
        text (str): The main string to be searched (T).
        pattern (str): The substring to be found (P).
        
    Returns:
        int: The starting index i of the match in T, or -1 if no match found.
    '''
    m = len(text)
    n = len(pattern)
    
    # We iterate through all possible starting positions i in the text.
    # The last possible start index is m - n.
    for i in range(m - n + 1): 
        x = 0  # x is the index for the pattern P
        
        # Compare characters in the current alignment
        while x < n and text[i + x] == pattern[x]: 
            x = x + 1
            
        # If x reached n, it means we compared all characters and found a full match
        if x == n:
            return i # Match found at starting index i
            
    return -1 # Search failed
    
# Test Cases
text1 = "abcabcaab"
pat1 = 'aab'
text2 = "aaaaabaaaaaaab"
pat2 = 'aaab'

print(f"Searching for '{pat1}' in '{text1}': Index {brute_force_search(text1, pat1)}")
print(f"Searching for '{pat2}' in '{text2}': Index {brute_force_search(text2, pat2)}")

```
